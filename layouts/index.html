Working style

How I approach documentation systems

01

Structure first

Metadata, taxonomy, and content models need to be designed as part of the system, not retrofitted after publication problems appear.

02

Quality gates that matter

Linting, review workflows, and release checks should reflect actual documentation risks, not just generic style preferences.

03

Portfolio, not archive

The site should frame writing as evidence of practice. Each page needs to help visitors understand the craft behind the output.
